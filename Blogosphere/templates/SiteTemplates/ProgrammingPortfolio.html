Common'Hood

>Unity developer: June - October 2021, July - October 2022

>As a developer, my tasks included:
>>Building and implementing a multiplayer sandbox mode
>>>Using the Mirror library for Unity, we ported all singe-player building features to a creative sandbox level, enabling friends to build and collaborate together

>>Creating a localization system
>>>All text, from dialogue to item flavor, had to be scraped and added to a spreadsheet
>>>Each line included the source of the text and a column for each language, including English
>>>Once the CSV spreadsheet was translated, the correct language was displayed to the player

>>Building and maintaining QoL features
>>>One feature automatically swapped to the next ingredient when constructing blueprints
>>>A developer tool I built displayed all 3d models, sorted by vertex count. This massively increased performance after several problem areas were identified
>>>On top of all my other tasks, we regularly cleared out the public buglist, fixing everything from pathfinding hiccups to texture glitches

SpaceRunners!

>Senior thesis project: Spring Quarter 2022

>>My senior thesis was a solo project in Godot, creating an android infinite runner
>>Everything, from programming to art, was done by me

<Playable builds>

>>Godot was chosen for two reasons:
>>>I agree with their open-source licensing on a moral level. Code is meant to be shared and improved by the entire community
>>>I wanted to train my adaptability, and learn a new system in a hands-on project. I had only ever used Unity until this point, and I wanted to experiment with a new language and engine

>>Systems I designed and built include:
>>>Turbo Boost, allowing the player to temporarily smash through asteroids and enemy Vipers
>>>Asteroids that destroy players, and explode on contact with lazers
>>>Lazers, to destroy foes and rocks
>>>Enemy ships, along with their AI
>>>Stat tracking, including kills, distance travelled, and number of times you bait an enemy into destroying an asteroid


>>One major advantage of creating a mobile game was it was much easier to find playtesters.
>>>I could hand over my phone, and get instant feedback
>>>This constant iteration made the game much more successful

Twin Rivers, Montana

>Solo Unity project: Summer-Fall 2020

>My first large scale solo unity project
>I designed the game, programmed it, and wrote the story
<Play in browser!>

>The dialogue system is an entirely custom implementation
>Branching dialogue trees were written in yarnspinner, then the JSON files were parsed by my system

>Some contextual lines needed previous game events to occur before the dialogue option appears
>>That context could include finding clues, or other conversations
>>This system allowed NPCs to respond directly to in-game events, increasing immersion and replayability

>In addition, important details from conversations are added to the player's journal



>Another major system I built was the clues
>>Scattered throughout the map are interactable clues.
>>After a clue is picked up, it is added to the player's journal
>>Creating new clues was a simple drag and drop

>I also created a map that tracked the player's location in real time
